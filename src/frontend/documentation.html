Overview

The index page provides control over the web scraping process through three primary actions:

  1. Create Request — Submit a URL and the number of chapters to scrape. You must also choose an output file extension. Available extensions indicate whether they support text and/or images. On submission, a request is created and you receive a Request ID. Better save it but it will as well be saved in cookies and shown at the bottom of the page.
  2. Add Website — Suggest a new website to be supported by submitting its URL. This does not start scraping, it queues your suggestion for future support.
  3. Download Result — Provide the Request ID from a completed job to download the results as a ZIP archive. You will get logs and result files there. After request is finished it will be available for repeated download for a fixed time.

The page also shows a list of pending requests. They are saved in cookies. They do not indicate whether request is finished or not - you need to make a request to find out. If you request some RID and it is successful or irredeemable, entry is removed.
